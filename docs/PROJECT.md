# 공도2 리뉴얼 기념 출석 이벤트 프로젝트

## 1. 프로젝트 개요

### 프로젝트명

공도2 리뉴얼 기념 출석 이벤트

### 주최

한양대학교 제49대 공과대학 학생회 건설준비위원회

### 목적

* 리뉴얼된 **공도2(공과대학 도서관 2)** 홍보
* 공도2 이용률 증가
* 공과대학 학생들의 공부 공간 활용 유도

### 대상

한양대학교 **공과대학 재학생**

### 예상 참여 인원

* 초기 참여: 약 150명
* 최종 5일 연속 출석자: 약 50명 이하 예상

# 3. 이벤트 참여 방법

1. 공도2 방문
2. 입구에 비치된 QR 스캔
3. 이벤트 웹사이트 접속
4. 최초 참여 시 정보 입력
5. GPS 위치 인증
6. 출석 완료

입력 정보

* 학과
* 학번
* 이름
* 전화번호

---

# 4. 출석 인정 조건

출석은 아래 조건을 모두 만족해야 인정된다.

1. 공도2 입구 QR 코드 스캔
2. GPS 위치 인증
3. 출석 가능 시간 내 인증
4. 하루 1회 출석 제한

GPS 조건

* 공도2 기준 반경 약 50m 이내

---

# 5. 연속 출석 규칙

이 이벤트는 **연속 출석 기반 이벤트**이다.

하루라도 출석하지 않을 경우 이벤트 참여 대상에서 제외된다.

예시

| 이름 | 16 | 17 | 18 | 19 | 20 |
| -- | -- | -- | -- | -- | -- |
| A  | O  | O  | O  | O  | O  |
| B  | O  | X  | 탈락 | 탈락 | 탈락 |

탈락자는 이후 데이터 관리 대상에서 제거된다.

# 8. 시스템 구성

이벤트는 **전용 웹사이트**를 통해 운영된다.

시스템 구조

QR → 웹사이트 접속
→ GPS 위치 확인
→ 사용자 인증
→ 출석 기록 저장
→ 출석 스탬프 확인

# 10. QR 코드 운영 방식

QR 코드는 **매일 변경된다.**

목적

QR 공유를 통한 부정 출석 방지

예시 URL

site.com/check?day=1
site.com/check?day=2
site.com/check?day=3
site.com/check?day=4
site.com/check?day=5

매일 다른 QR을 출력하여 공도2 입구에 비치한다.

웹 환경에서는 기기 일련번호와 같은 고유 식별자를 얻을 수 없기 때문에 아래 방식으로 부정 출석을 방지한다.

### 3. 하루 1회 출석 제한

중복 출석 방지

### 5. 이상 행동 탐지

관리자 페이지에서 아래 사항을 모니터링한다.

* 동일 IP에서 다수 계정 출석
* 동일 위치 데이터 반복 출석
* 짧은 시간 내 다수 출석 시도

의심 사례는 관리자 검토 후 제외할 수 있다.
